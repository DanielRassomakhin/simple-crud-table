# Simple CRUD Table

CRUD-приложение для управления заявителями на Vue 3 + Pinia + TypeScript с имитацией бэкенда через MSW.

## Стек

| Технология         | Версия | Роль                                       |
| ------------------ | ------ | ------------------------------------------ |
| Vue 3              | ^3.5   | UI-фреймворк, Composition API              |
| TypeScript         | ~5.9   | Статическая типизация                      |
| Zod                | ^4     | Схемы и runtime-валидация                  |
| Pinia              | ^3     | Стейт-менеджмент                           |
| Vuetify            | ^4     | UI-компоненты                              |
| Vue Router         | ^5     | Роутинг                                    |
| Axios              | ^1     | HTTP-клиент                                |
| MSW                | ^2     | Fake API (Mock Service Worker)             |
| VueUse             | ^14    | Утилиты (`useDebounceFn`, `useIntervalFn`) |
| Maska              | ^2.1   | Маска ввода телефона                       |
| vue-toastification | ^2     | Toast-уведомления                          |
| Vite               | ^7     | Сборщик                                    |

## Функциональность

### Таблица заявителей

- Серверная пагинация через `VDataTableServer` (10 записей по умолчанию, можно изменить)
- Сортировка по колонкам «ФИО» и «Создан» — клик по заголовку

### Поиск и фильтрация

- Поле «Поиск по ФИО» — частичное совпадение с debounce 400 мс
- Поиск нечувствителен к регистру и к букве Ё (`«петр»` найдёт `«Пётр»`, `«е»` найдёт `«ё»`)
- Кнопка «Показать фильтры» раскрывает дополнительный фильтр по статусу

### CRUD

| Действие      | Как выполнить                                             |
| ------------- | --------------------------------------------------------- |
| Создать       | Кнопка «Добавить заявителя» → заполнить форму → «Создать» |
| Редактировать | Кнопка «Редактировать» в строке таблицы                   |
| Удалить       | Кнопка «Удалить» в строке → подтвердить в диалоге         |

После каждой операции появляется toast-уведомление с результатом.

### Форма

- **ФИО** — обязательно, три слова через пробел, только буквы (кириллица и латиница)
- **Телефон** — маска `+7 ### ###-##-##`, ровно 11 цифр; `+7` подставляется по умолчанию
- **Статус** — выбор из `Новый / В работе / Завершён`
- Кнопка подтверждения неактивна, пока не заполнены все поля без ошибок
- Inline-валидация: ошибки появляются и исчезают прямо при вводе

### Live Update

Кнопка «Live update» в шапке включает автоматическое обновление статусов раз в 2 секунды — имитация работы реального бэкенда в реальном времени.

## Архитектура

Проект построен по принципам **Domain-Driven Design (DDD)** с разделением на слои:

```
src/
├── shared/                          # Общие утилиты (без привязки к домену)
│   └── validation/
│       ├── index.ts                 # fullNameSchema, phoneRuSchema — переиспользуемые Zod-валидаторы
│       └── useFormValidation.ts     # Composable для валидации любой Zod-схемы
│
├── domain/                          # Доменный слой
│   └── applicants/
│       ├── applicant.model.ts       # Zod-схема сущности + типы (Applicant, ApplicantStatus)
│       └── applicant.form.ts        # applicantFormSchema + ApplicantFormInput/Output
│
├── application/                     # Слой приложения
│   └── applicants/
│       └── applicants.store.ts      # Pinia-стор: состояние + бизнес-логика
│
├── infrastructure/                  # Инфраструктурный слой
│   ├── api/
│   │   ├── httpClient.ts            # Axios-инстанс
│   │   └── api.ts                   # Функции запросов к API
│   └── msw/
│       ├── browser.ts               # Инициализация MSW worker
│       ├── handlers/                # Обработчики HTTP-запросов (fake API)
│       └── data/                    # In-memory база данных
│
├── presentation/                    # Презентационный слой
│   ├── applicants/                  # Фича «Заявители»
│   │   ├── ApplicantsPage.vue       # Страница-оркестратор
│   │   ├── ApplicantsTable.vue      # Таблица
│   │   ├── ApplicantsFilters.vue    # Поиск + фильтр по статусу
│   │   ├── ApplicantsFormDialog.vue # Форма создания/редактирования
│   │   ├── ApplicantsHeader.vue     # Шапка с кнопками
│   │   └── composables/
│   │       └── useApplicantsTable.ts  # Заголовки, данные, сеттеры таблицы
│   └── components/                  # Переиспользуемые UI-компоненты
```

## Ключевые технические решения

### TypeScript + Zod

TypeScript — базовая типизация на уровне компилятора. Zod дополняет его **runtime-валидацией**: схема описывается один раз, из неё же выводятся TypeScript-типы через `z.infer`. Это исключает дублирование типов между слоями.

Валидация форм разделена по слоям:

- `shared/validation/` — универсальные схемы (`fullNameSchema`, `phoneRuSchema`), не зависящие от домена
- `domain/applicants/applicant.form.ts` — `applicantFormSchema`, собирающая доменные правила из shared
- Zod-трансформы (`z.transform`) нормализуют данные прямо при валидации (обрезка пробелов, извлечение цифр телефона)

### useFormValidation

Composable `useFormValidation(schema)` инкапсулирует всю логику валидации формы:

```ts
const { errors, hasErrors, validate, validateField, clearErrors } =
  useFormValidation(applicantFormSchema);

// Валидация одного поля при вводе
validateField("phone", form.phone);

// Полная валидация при сабмите — возвращает трансформированные данные или null
const result = validate(form);
```

Компонент формы не содержит ни одной строки валидационной логики — только UI и управление состоянием диалога.

### Pinia

Централизованное хранилище для состояния таблицы (`items`, `total`, `page`, `search`, `status`, `sortBy`). Все toast-уведомления инициируются прямо из стора — компоненты не знают о результатах операций.

`mutationLoading` отделён от `loading` — это позволяет показывать progress-индикатор при CRUD-операциях, не перекрывая данные таблицы при live-обновлениях.

### MSW (Mock Service Worker)

Перехватывает реальные HTTP-запросы на уровне Service Worker. Axios не знает о подмене — код приложения идентичен коду с настоящим API. Fake API реализует серверную пагинацию, поиск с нормализацией Ё→Е, фильтрацию и сортировку.

### VueUse

`useDebounceFn` — debounce поиска без ручного `setTimeout`. `useIntervalFn` — live-обновления с автоматической очисткой таймера при `onBeforeUnmount`.

### Vuetify 4 (a-la-carte)

Подключаются только нужные компоненты (`VDataTableServer`, `VBtn`, `VTextField` и т.д.) + MDI-иконки через `@mdi/font`.

## Что можно улучшить при наличии времени

### Качество кода

- **Unit-тесты** для стора и composable (Vitest + Vue Test Utils) — `useFormValidation` уже легко тестируется изолированно
- **E2e-тесты** (Playwright) — сценарии создания, редактирования, удаления
- **Обработка ошибок** — добавить inline-состояние ошибки в таблице с кнопкой retry вместо только toast

### Архитектура

- **Repository pattern** — выделить слой репозиториев между стором и API-функциями, чтобы стор не зависел от Axios напрямую
- **Use Cases** — вынести бизнес-логику (`liveTick`, агрегацию параметров запроса) из стора в отдельные сервисы application-слоя
- **Глобальный composable для управления модалками** - для управления модалками использовать глобальный composable, а не компоненты и обращаться через хук
- **Сделать инкапсуляцию api**
- **Дедупликация данных**

### UX / Функциональность

- **URL-синхронизация** — сохранять фильтры, сортировку и страницу в query string для шаринга ссылок
- **Skeleton loading** — placeholder-строки при первой загрузке
- **Bulk actions** — выделение нескольких строк и массовое удаление/изменение статуса

### UI

- **UI** - сделать отзывчивым и красивым
